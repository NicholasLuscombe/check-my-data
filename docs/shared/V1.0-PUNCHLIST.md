### C1. Table-content design-call triage — **resolved S280**
The cluster of BANKED items here turned out already-ruled at source:
- **Finding-vs-significance vocabulary.** The first proposed lock (`Finding` = effect-size descriptor)
  was killed at the S280 catalogue: the descriptor content is heterogeneous (direction labels, ratio
  bands, κ deviations), not effect size. The S220 inferential-vs-descriptive rule already governs why a
  benign word can sit beside a significant p — several of these surfaces (Within-Row Variance, LOESS,
  Kurtosis) are deliberately descriptive, with rationale in METHODOLOGY. **The live gap was the two
  cards with that shape and no explanatory caption** (Selective Noise, Cross-Cond). Fixed S280 with a
  caption above each evidence table, matching the existing Kurtosis (`:72`) / Runs (`:234`) precedent.
- **Raw-vs-adjusted-p display** (Runs Table 1, Mahalanobis) — already ruled: Mahalanobis is an S219
  principled exemption (displays raw distance + the threshold line carries the corrected cut); Runs
  per-pair gates on `adjP` per S218. No edit owed.
- **8a "Pass" column** (ValueFrequency) — confirmed a detection-method label (which scan pass fired),
  not a verdict descriptor (S280 audit Note B). Correctly excluded from the descriptor convention.
  **Superseded S282:** the column was dropped entirely — it is invariant within a fixture (carries one
  bit, which pass drove, differing only across fixtures), lifted to a `drivingPass`-read heading note
  (`ef2d773`). This also cleared the DS04 table overflow. The no-Finding-column half of 8a is parked #39.

### C2. Finding-column-position convention — **resolved S280** (already-ruled S219)
The suite split (ColumnGoF/Entropy at position 2; Kurtosis/WithinRowVariance/CrossCond rightmost) is a
**recorded, reasoned S219 exception**, not an open convention. ColumnGoF/Entropy keep `Finding` at
position 2 because of a DataTable font constraint — `Finding` only renders sans-serif inside
`identifierColumns=2`; moving it rightmost would force `FF.MONO` or a DataTable contract-widening the
S217 zero-widening principle forbids. The walk's 10a Direction-column row is homed by that ruling.
No reorder owed.
